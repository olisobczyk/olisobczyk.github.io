Offshore Wind Farm Techno-Economic Assessment ↗

Python | MATLAB | Data Science | ERA5 Data | Financial Modeling
Impact: €5.46B NPV, 17.28% IRR

The Challenge

Assess technical feasibility and economic viability of a large-scale offshore wind farm in Greek waters—a region with significant wind potential but emerging regulatory framework and limited precedent. Required rigorous validation of wake modeling under Mediterranean conditions and comprehensive financial analysis accounting for capital constraints and market conditions.

The Solution

Conducted end-to-end techno-economic analysis combining meteorological data science, advanced wake modeling, and financial engineering. Processed 2.3GB of ERA5 hourly wind data (u/v velocity components at 10m and 100m heights) using Python, then implemented Jensen/PARK wake model in MATLAB to simulate turbine interactions and power generation across the full year.

Key Achievements

  • Data Engineering: Extracted, processed, and validated 2.3GB ERA5 meteorological dataset; converted NetCDF files to structured CSV using Python (NumPy, xarray)
  • Scientific Modeling: Implemented Jensen/PARK wake model from first principles, accounting for wind shear, air density variation, and turbine thrust dynamics
  • Validation: Proved model reliability for Greek maritime conditions; identified seasonal variability and wake loss patterns (14.03% annual mean)
  • Economic Analysis: Computed NPV (€5.46B), IRR (17.28%), and LCOE (€134/MWh) with sensitivity analysis identifying capital optimization as primary cost driver
  • Stakeholder Assessment: Analyzed ecosystem of actors and global precedents for successful project deployment
  • Key Finding: Project is both technically viable and financially attractive; 9,216 GWh annual energy production with 55.31% capacity factor

My Role

Independent researcher. Designed research methodology, extracted and processed all meteorological data, coded and debugged wake model, performed economic analysis, and synthesized interdisciplinary findings. 3-month project completed solo.

AI-Powered Python Financial Assistant ↗

Python | NeuralIntents | pandas-datareader | mplfinance | pickle
Impact: End-to-end conversational AI assistant for real-time portfolio management and market visualisation

The Challenge

Build an intelligent, conversational interface for personal portfolio management and stock market analysis — without hardcoding every possible user phrase. The assistant needed to understand natural language intent, fetch live financial data, and persist portfolio state across sessions, all from the command line.

The Solution

Developed a modular CLI application using the NeuralIntents library to train a custom intent-classification model on a curated set of financial conversation patterns. Integrated pandas-datareader and yfinance (as an automatic fallback) for live Yahoo Finance data, and mplfinance for styled candlestick charting. Portfolio state is serialised with pickle for persistence between runs.

Key Achievements

  • NLP Intent Engine: Trained a neural network on 8 intent categories and ~60 labelled patterns covering greetings, charting, portfolio CRUD, valuation, and performance analysis
  • Live Market Data: Fetches real-time OHLCV data from Yahoo Finance with automatic yfinance fallback to handle API instability — transparent to the user
  • Portfolio Analytics: Real-time total valuation across all holdings; historical gain/loss comparison against any user-specified date with absolute and relative return metrics
  • Candlestick Charting: Styled dark-theme charts with custom bull/bear colours and volume overlay via mplfinance
  • Clean Architecture: Separated concerns across four modules (NLP layer, intent handlers, market data, portfolio persistence) for maintainability and extensibility
  • Model Caching: Saves trained model weights to disk; --load flag skips retraining on subsequent runs for 5–10× faster startup

My Role

Solo developer. Reconstructed and extended the project from a tutorial transcript, applied production-quality engineering practices (type hints, error handling, fallback data sources, input validation), and structured the codebase for long-term maintainability.
